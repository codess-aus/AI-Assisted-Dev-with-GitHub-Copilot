Welcome & AI-Assisted Development Landscape

The Rise of AI-Assisted Development

Artificial intelligence is fundamentally reshaping how we write code. GitHub Copilot represents a watershed moment in software development—a tool that brings AI pair programming directly into your IDE. Whether you're maintaining legacy systems, building new features, or exploring cutting-edge architectures, understanding how to work effectively with AI-assisted code generation is becoming essential.

This workshop is designed to take you from "What's Copilot?" to "How do I integrate this into my daily workflow?" We'll explore its capabilities, acknowledge its limitations, and equip you with practical techniques to use Copilot responsibly and productively.

Learning Objectives

  • Set expectations for the workshop and understand what AI-assisted development means
  • Recognize Copilot's capabilities as a code suggestion and generation tool
  • Understand Copilot's limitations and when to rely on human judgment
  • Learn typical workflows for using Copilot effectively in daily development
  • Differentiate between Copilot as an autocomplete tool versus as a true pair programmer

What is GitHub Copilot?

GitHub Copilot is an AI coding assistant powered by OpenAI's large language models (specifically trained on public code repositories). It integrates directly into VS Code, Visual Studio, JetBrains IDEs, and other development environments, providing real-time code suggestions as you type.

Key Capabilities:

  • Generate multi-line code completions from function signatures or comments
  • Write unit tests and documentation
  • Suggest refactorings and improvements
  • Support multiple programming languages (Python, JavaScript, TypeScript, C#, Java, Go, Rust, and many more)
  • Work with both inline completions and chat-based interactions (via Copilot Chat)

Key Limitations to Keep in Mind

Copilot is incredibly powerful, but it's not a silver bullet. Understanding its constraints will help you use it more effectively:

  • Hallucinations: Copilot can generate plausible-sounding but incorrect code, including references to non-existent functions or libraries
  • Limited reasoning: It works best with clear context; complex multi-step logic may require human guidance
  • Security awareness: Copilot isn't trained to avoid all security vulnerabilities; you must review and test carefully
  • Outdated knowledge: Training data has a cutoff date; newer APIs or frameworks may not be well-represented
  • Context window: Copilot has a limited "view" of your codebase; large files or complex architectures can confuse it

Copilot as Autocomplete vs. True Pair Programming

It's important to understand the spectrum of Copilot usage:

Autocomplete Mode: You type a few characters, and Copilot suggests the next word or line. This is fast and useful for routine coding patterns, but you're still driving the conversation.

Pair Programming Mode: You write a detailed comment describing what you want to build, and Copilot generates substantial code blocks or entire functions. You then review, refactor, and test. This mode is more powerful but requires more critical thinking on your part.

Agent Mode: Copilot takes more autonomy, making multi-step decisions to solve larger problems. This is the frontier of AI-assisted development and brings both exciting possibilities and new responsibilities.

Your Mindset Going Forward

The most successful Copilot users adopt a few core principles:

  • Stay in control: You're the senior developer. Copilot is your junior developer. Review, question, and improve every suggestion.
  • Be explicit: The better your prompts (via comments, variable names, and docstrings), the better Copilot's output.
  • Test everything: Copilot-generated code should pass the same scrutiny as any other code—automated tests, security reviews, code reviews.
  • Iterate together: If Copilot's first suggestion isn't right, refine your prompt and try again.

Lab 1: Getting Started with Copilot

In this hands-on lab, you'll set up Copilot and take it for a spin. We'll work in both Python and C# to show that Copilot's capabilities span multiple languages.

Setup

  • Ensure Copilot extension is installed in VS Code (GitHub Copilot by GitHub)
  • Sign in with your GitHub account

Exercise 1: Simple Completion

Choose your preferred language and let Copilot suggest the implementation:

Create a new file: fizzbuzz.py

Step 1: Write a comment and function signature:

# Write a function that takes a number and returns "Fizz" if divisible by 3, "Buzz" if divisible by 5, and "FizzBuzz" if divisible by both. Otherwise, return the number as a string.
def fizzbuzz(n):
    pass

Now press Ctrl+Enter (or Cmd+Enter on Mac) to trigger Copilot's suggestion panel. Accept the suggestion or iterate by providing more detail in your comment.

Step 2: Once Copilot completes your function, add this test code at the bottom of your file:

# Test your function
if __name__ == "__main__":
    for i in range(1, 16):
        print(f"{i}: {fizzbuzz(i)}")

Run it: python fizzbuzz.py

Expected output: You should see numbers 1-15 with FizzBuzz logic applied (3 → "Fizz", 5 → "Buzz", 15 → "FizzBuzz", etc.)

Create a new file: FizzBuzz.cs

Step 1: Write the class structure with a comment and method signature:

// Implement FizzBuzz logic as a method in a class
using System;

public class FizzBuzzSolver
{
    // Write a method that takes a number and returns "Fizz" if divisible by 3,
    // "Buzz" if divisible by 5, and "FizzBuzz" if divisible by both.
    // Otherwise, return the number as a string.
    public string Solve(int n)
    {
        return n.ToString();
    }
}

Now press Ctrl+Enter (or Cmd+Enter on Mac) to trigger Copilot's suggestion panel. Notice how it adapts to C# syntax and conventions.

Step 2: Once Copilot completes your method, add this Main method to test it (inside the class, after the Solve method):

    static void Main()
    {
        var solver = new FizzBuzzSolver();
        for (int i = 1; i <= 15; i++)
        {
            Console.WriteLine($"{i}: {solver.Solve(i)}");
        }
    }

Run it: dotnet run or compile with csc FizzBuzz.cs then run ./FizzBuzz

Expected output: Numbers 1-15 with FizzBuzz logic applied

Create a new file: fizzbuzz.js

Step 1: Write a comment and function signature:

// Implement FizzBuzz logic in JavaScript
// Return "Fizz" for multiples of 3, "Buzz" for 5, "FizzBuzz" for both,
// otherwise return the number as a string.
function fizzbuzz(n) {
    return n.toString();
}

Now press Ctrl+Enter (or Cmd+Enter on Mac) to trigger Copilot's suggestion panel. Notice how it adapts to JavaScript idioms (strict equality, modulo checks).

Step 2: Once Copilot completes your function, add this test code at the bottom of your file:

// Test your function
for (let i = 1; i <= 15; i++) {
    console.log(`${i}: ${fizzbuzz(i)}`);
}

Run it: node fizzbuzz.js

Expected output: Numbers 1-15 with FizzBuzz logic applied

Exercise 2: Iteration and Refinement

Now let's enhance your function to handle edge cases. This demonstrates how to iterate with Copilot—improve your comment with more detailed requirements, and Copilot generates better code.

Step 1: Clear your fizzbuzz.py file and start fresh with an enhanced comment. Here's your starting point:

# Write a function that takes a number and returns "Fizz" if divisible by 3, "Buzz" if divisible by 5, and "FizzBuzz" if divisible by both.
# Otherwise, return the number as a string.
# Raise a ValueError if the number is negative.
def fizzbuzz(n):
    pass

Position your cursor at the end of the `pass` statement and press Ctrl+Enter to trigger Copilot. It will now generate code that includes error handling for negative numbers.

Step 2: Replace your entire test code section with this complete version (delete the old test code first, then paste this):


if __name__ == "__main__":
    print("=== Normal Cases (1-15) ===")
    for i in range(1, 16):
        print(f"{i}: {fizzbuzz(i)}")
    
    print("\n=== Edge Cases ===")
    
    # Test negative number (should raise an error)
    print("Testing fizzbuzz(-5):")
    try:
        result = fizzbuzz(-5)
        print(f"Result: {result}")
    except ValueError as e:
        print(f"✓ Caught error: {e}")
    
    # Test zero (should work)
    print(f"\nfizzbuzz(0): {fizzbuzz(0)}")

Run it: python fizzbuzz.py

Expected output: You should see the FizzBuzz sequence for 1-15, then a section showing the error handling working (with a ✓ checkmark), and fizzbuzz(0) returning "FizzBuzz".

Step 1: Clear your FizzBuzz.cs file and start fresh with an enhanced comment. Here's your starting point:
